Block vs. Inline vs. Semantic Elements

1. The P Tag (Semantic Block Element)

<p> (Paragraph) is a **Block-Level** element.

It is used for blocks of text, and by default, the browser adds space (margins) above and below it.

2. The DIV Tag (Generic Block Element)

<div> (Division) is also a **Block-Level** element. It is a general-purpose container for grouping other elements.

By default, it takes up the entire width available and forces a line break before and after, just like $\text{p}$.

3. The SPAN Tag (Generic Inline Element)

<span> is an **Inline** element. It is a general-purpose container used to style or manipulate a small part of content *within* a larger block.

It **does not** force a line break, and it only takes up as much width as its content needs.

Summary of Layout Behavior

**BLOCK ELEMENTS** ($\text{p, div, h1, li, etc.}$):

  • Start on a new line.
  • Take up the full width available.
  • Stack vertically.

**INLINE ELEMENTS** ($\text{span, a, em, strong, etc.}$):

  • Do not start on a new line.
  • Only take up as much width as their content.
  • Flow horizontally with the text.
